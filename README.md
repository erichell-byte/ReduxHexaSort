# ReduxPuzzle  

Отчёт  
Время на задачу: ≈ 6 часов (добавление слоёв архитектуры, DI, бустеров, сохранений)  

Архитектура  
-Hexa.Core – стейт-машина, загрузка сцен, Zenject-установщики.  
-Hexa.Domain – чистая логика: гекса-решётка, стопки, правила победы/проигрыша, команды бустеров, undo-состояния.  
-Hexa.Infrastructure – сервисы анимации, ввода, сохранения (PlayerPrefs), общий инпут.  
-Hexa.Gameplay.Services – размещение башен, цепные реакции, генератор поля, колода игрока, drag-сервис.  
-Hexa.Presentation – презентеры (HUD/Menu/GameOver/Booster) + UI-модели/вью, BoardController, PlayerTower, MenuView и т.п.  
  
Dependency Injection через Zenject, фабрики для префабов, юнитестов нет, асинхронность через UniTask, анимации через DOTween.  

Известные проблемы / улучшения  

Размер поля – зависит только от LevelConfig.HexCount, изменяется вручную.  
Бустер “Уничтожить стопку” можно сделать яснее: сейчас надо выбрать клетку после нажатия кнопки.  
BoardController перегружен (управляет undo, спавном, логикой бустеров) – стоит дробить.  
Тесты / валидация отсутствуют: любое изменение требует ручной проверки в Unity.  
Анимации основаны только на DOTween. При желании можно вынести конфигурацию анимаций в ScriptableObject.  

ИИ использовал для рефакторинга некоторых классов и оценки слабых мест.
